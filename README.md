# ExpCard Converter

试验卡数据提取工具，将 Excel 试验卡中的逻辑条件快速提取并转换为 Markdown 格式，支持导出为 Word 文档。

## 核心功能

- **智能逻辑提取** — 自动识别"与"、"且"、"或"、"或取反"等逻辑运算符
- **双模式转换** — 支持子序号模式（x.y 格式）和纯数字模式（列嵌套）
- **特殊分隔符处理** — 自动检测并转换"或延时"、"与延时"等特殊分隔符
- **自定义检测规则** — 支持配置行识别规则，适配不同格式的 Excel 表格
- **Web UI 配置** — 可视化配置界面，无需手动修改代码
- **Word 导出** — 通过 pandoc 将 Markdown 转换为 `.docx` 格式
- **多主题支持** — 浅色、深色、毛玻璃三种 UI 主题
- **大纲导航** — 可折叠的大纲树，快速定位内容

## 快速开始

### 方式一：使用启动脚本（推荐）

双击 `start.ps1` 或 `start.bat`，选择 `[1] 启动服务`，其余交给脚本处理。

**启动脚本会自动完成以下操作：**

- 检测配置文件、Word 转换工具（pandoc）是否存在
- 检测端口 3210 是否被占用，若被占用则自动关闭旧进程释放端口
- 在后台启动服务（不占用终端窗口）
- 等待服务就绪后自动打开浏览器

你只需拖入 Excel 文件、选择工作表、点击转换，无需手动操作端口和浏览器。

### 方式二：手动启动

```bash
cd dcs-expcard-tool-nodejs
npm install
npm start
```

服务启动后手动访问 `http://localhost:3210`

## 转换逻辑详解

### 整体流程

```
Excel 表格 (xlsx)
    │
    ▼
┌──────────────────────────────────────────────┐
│  1. 读取 Sheet 数据，按行遍历                   │
│  2. 跳过表头行（序号、条件确认）                 │
│  3. 识别中文数字标题（一、二、三、）→ ## 标题    │
│  4. 识别段落标题（试验条件/试验恢复/结论/存在问题）│
│  5. 识别子标题（1.1 试验内容（与））→ ### 标题   │
│  6. 自动判断模式：x.y 格式 vs 纯数字             │
│  7. 按层级拆分逻辑，拼接输出                     │
└──────────────────────────────────────────────┘
    │
    ▼
Markdown 输出
```

### 行识别规则

| 行类型 | 识别条件 | 处理方式 |
|--------|----------|----------|
| 表头行 | A列 = `序号` 或 `条件确认`，或 A=`序号` 且 B 含 `条件/内容` | 跳过 |
| 中文数字标题 | A列匹配 `/^[一二三四五六七八九十]+、/` | 输出 `## 标题` |
| 段落标题 | A列 ∈ `["试验条件", "试验恢复", "结论", "存在问题"]` | 设为跳过区间 |
| 内容起始 | A列 = `试验内容` | 开始处理内容行 |
| 子标题 | A列匹配 `/^\d+\.[\u4e00-\u9fa5]/`（如 `1.1 试验条件（与）`） | 输出 `### 标题`，提取括号内逻辑 |
| 纯数字行 | A列匹配 `/^\d+$/`（如 `1`、`2`） | 纯数字模式处理 |
| 子项行 | A列匹配 `/^\d+\.\d+$/`（如 `1.1`、`1.2`） | 子序号模式处理 |

### 模式判断

| 条件 | 模式 | 入口方法 |
|------|------|----------|
| A 列含 `x.y` 格式（如 `1.1`） | 子序号模式 | `convert()` → `_processLevel1()` / `_processLevel2()` |
| A 列为纯数字（如 `1`、`2`） | 纯数字模式 | `convertNew()` → `_processNewItem()` |

### 模式一：子序号模式（x.y 格式）

适用于 A 列使用 `1.1`、`1.2`、`2.1` 等编号的表格。编号本身表达层级关系。

#### Excel 结构示例

| A (编号) | B (逻辑) | C (内容/逻辑) | D (子逻辑) | E (内容) |
|----------|----------|---------------|------------|----------|
| 试验内容 | | | | |
| 1.1 A磨润滑油条件满足（与） | | | | |
| | 与 | 条件A | | |
| | | 或 | | 子条件1 |
| | | | 或 | 子条件2 |
| | 或 | 条件B | | |

#### 处理流程

```
子标题行 "1.1 A磨润滑油条件满足（与）"
    │  识别为子标题（A列匹配 x.y + 中文）
    │  提取括号内逻辑（与/或）→ 存为 currentSubTitleLogic
    │  输出: "### 1.1 A磨润滑油条件满足（与）"
    │
    ▼
Level 1: 主条件行（x.y 行的下一行，B列有内容）
    │  检查下一行 A 列是否为 x.y 子项
    │  ├─ 是 → 有子条件，需要判断逻辑
    │  │   ├─ B列是逻辑运算符 → 用 B 列逻辑
    │  │   └─ B列不是逻辑运算符 → 用下一行的 B 列逻辑
    │  └─ 否 → 直接输出
    │
    │  输出: "1.2.1 A磨润滑油条件满足（且）"
    │  括号逻辑 = currentSubTitleLogic 或 下一行 B 列逻辑
    │
    ▼
Level 2: 子条件行（x.y 子项，C列判断）
    │  检查 C 列内容：
    │  ├─ C 列是逻辑运算符 + D 列是逻辑运算符 → Level 3 分组
    │  ├─ C 列是逻辑运算符 + D 列不是逻辑运算符 → Level 2 分组
    │  ├─ C 列不是逻辑运算符 → 直接输出
    │  └─ B 列是逻辑运算符 → 收集 C 列内容
    │
    │  输出: "   - 条件A"
    │
    ▼
Level 3: 嵌套分组（C列 + D列都是逻辑运算符）
    │  按 D 列逻辑分组，收集 E 列内容
    │  同组内多项用 D 列逻辑连接
    │  多组之间用"或"连接
    │  单组多项用括号包裹
    │
    │  输出: "   - （子条件1 或 子条件2）"
```

#### 输出示例

```markdown
## 一、试验内容

### 1.1 A磨润滑油条件满足（且）

1.2.1 A磨润滑油条件满足（且）
   - 条件A 或 子条件1 或 子条件2
   - 条件B
```

### 模式二：纯数字模式

适用于 A 列使用纯数字 `1`、`2`、`3`，通过列嵌套表达层级关系。每一列代表一层嵌套。

#### Excel 结构示例

| A (编号) | B (逻辑) | C (逻辑/内容) | D (逻辑/内容) | E (内容) |
|----------|----------|---------------|---------------|----------|
| 试验内容 | | | | |
| 1 | 与 | 条件A | | |
| | | 或 | | 子条件1 |
| | | | 或 | 子条件2 |
| | 或 | 条件B | | |

#### 处理流程

```
纯数字行 "1"（A 列）
    │
    ├─ 下一行 A 列是 x.y 格式？→ 切换到子序号模式处理
    │
    └─ B 列是逻辑运算符？
        ├─ 否 → 无嵌套，直接输出: "1. 条件A"
        └─ 是 → 有嵌套，进入 Level 2
            │
            ▼
Level 2: B 列逻辑 + C 列判断
    │  遍历后续行，检查 C 列：
    │  ├─ C 列是逻辑运算符 → 进入 Level 3
    │  └─ C 列不是逻辑运算符 → 收集内容，用 B 列逻辑连接
    │      找到下一个逻辑运算符时停止收集
    │
    │  输出: "1. 条件A 或 子条件1 或 子条件2"
    │
    ▼
Level 3: C 列逻辑 + D 列判断
    │  遍历后续行，检查 D 列：
    │  ├─ D 列是逻辑运算符 → 进入 Level 4
    │  └─ D 列不是逻辑运算符 → 收集内容，用 C 列逻辑连接
    │      多项用括号包裹: "（子条件1 或 子条件2）"
    │
    ▼
Level 4: D 列逻辑 + E 列内容
    │  收集后续行的 E 列内容，用 D 列逻辑连接
    │  多项用括号包裹: "（子条件1 或 子条件2）"
    │
    ▼
输出最终结果
```

#### 输出示例

```markdown
## 一、试验内容

1. 条件A 或 （子条件1 或 子条件2）
2. 条件B
```

### 逻辑运算符映射

| Excel 值 | 输出值 | 说明 |
|---------|-------|------|
| 与 | 且 | 逻辑与 |
| 且 | 且 | 逻辑与（同义词） |
| 或 | 或 | 逻辑或 |
| 或取反 | （或取反） | 逻辑或取反，输出时加括号 |

### 特殊分隔符

采用**前缀匹配**，用于处理"或延时xxS"等无法正常拼接的逻辑。

| 特殊值 | 转换为 | 匹配方式 | 示例 |
|-------|-------|---------|------|
| 或延时 | 或 | 前缀匹配 | `或延时720s` → `或` |
| 与延时 | 且 | 前缀匹配 | `与延时300s` → `且` |

特殊分隔符会被记录位置，用于后续标记警告（提示用户哪些位置存在特殊分隔符）。

### 跳过规则

以下行会被跳过，不参与转换：

- **表头行**：A列 = `序号` 或 `条件确认`
- **段落标题**：A列 ∈ `["试验条件", "试验恢复", "结论", "存在问题"]`
- **区间跳过**：`试验条件` 之后、`试验内容` 之前的所有行（`skipSection` 标记）

### 嵌套括号规则

| 层级 | 条件 | 输出格式 | 示例 |
|------|------|---------|------|
| Level 1 | 有子条件 | `序号. 内容（逻辑）` | `1. 条件A（且）` |
| Level 2 单项 | 无分组 | `   - 内容` | `- 条件A` |
| Level 2 分组 | C列是逻辑运算符 | `   - （内容1 或 内容2）` | `- （子条件1 或 子条件2）` |
| Level 3 | D列是逻辑运算符 | `   - （（内容1 或 内容2） 或 内容3）` | `- （（A 或 B） 或 C）` |
| Level 4 | E列多行 | `   - （（（内容1 或 内容2） 或 内容3） 或 内容4）` | `- （（（A 或 B） 或 C） 或 D）` |

**括号嵌套逻辑**：
- 同一层级内多项 → 用逻辑运算符连接，外加括号
- 不同层级之间 → 外层括号包裹内层
- 最终输出保持层级嵌套关系

## 配置文件

配置文件 `config.js` 位于程序目录，可在启动前修改。也可通过 Web UI 的 ⚙️ 设置按钮进行可视化配置。

```javascript
const EXPCARD_CONFIG = {

  // 逻辑运算符映射
  // 格式：{ 键: 值 }，键为Excel中的值，值为输出时使用的值
  LOGIC_OPERATORS: {
    与: "且",
    且: "且",
    或: "或",
    或取反: "或取反",
  },

  // 特殊分隔符映射（前缀匹配）
  // 格式：{ 键: 值 }，键为特殊值前缀，值为转换后的输出
  SPECIAL_SEPARATORS: {
    或延时: "或",
    与延时: "且",
  },

  // 行识别规则
  // 支持三种匹配类型：exact(精确匹配)、regex(正则匹配)、startsWith(前缀匹配)
  ROW_PATTERNS: {
    // 表头行：A列精确匹配 values 中的值时，跳过该行
    header: { type: "exact", values: ["序号", "条件确认"] },

    // 中文数字标题：A列匹配正则时，输出为 ## 标题
    chineseTitle: { type: "regex", pattern: "^[一二三四五六七八九十]+、" },

    // 子标题：A列匹配正则时，输出为 ### 标题
    subTitle: { type: "regex", pattern: "^\\d+\\.[\\u4e00-\\u9fa5]" },

    // 纯数字行：A列匹配正则时，作为一级内容处理
    pureNumber: { type: "regex", pattern: "^\\d+$" },

    // 子项行：A列匹配正则时，作为二级内容处理
    subItem: { type: "regex", pattern: "^\\d+\\.\\d+$" },

    // 段落标题：A列精确匹配 values 中的值时，跳过该行及后续内容
    paragraphTitle: { type: "exact", values: ["试验条件", "试验恢复", "结论", "存在问题"] },

    // 内容起始标记：A列精确匹配 values 中的值时，开始处理后续内容行
    contentStart: { type: "exact", values: ["试验内容"] },

    // 跳过区间开始：A列精确匹配 values 中的值时，开始跳过内容
    skipSectionStart: { type: "exact", values: ["试验条件"] },
  }
};
```

### 配置方式

**方式一：修改配置文件**
直接编辑 `config.js`，保存后需重启服务生效。

**方式二：Web UI 配置**
1. 启动服务后访问 http://localhost:3210
2. 点击右上角 ⚙️ 按钮打开配置面板
3. 修改配置后点击"保存配置"，无需重启服务

**方式三：导入/导出配置**
- 点击"导出配置"可将当前配置保存为 JSON 文件
- 点击"导入配置"可从 JSON 文件导入配置

## Excel 文件规范

### 格式要求

- 格式：`.xlsx`（不支持 `.xls`）
- 编码：UTF-8
- 首行：表头行（自动跳过）

### 两种模式对比

| 特性 | 子序号模式 | 纯数字模式 |
|------|-----------|-----------|
| A 列编号 | `x.y` 格式（如 `1.1`） | 纯数字（如 `1`） |
| 层级表达 | 编号本身表达层级 | 列嵌套表达层级 |
| 适用场景 | 层级清晰的表格 | 宽表格式 |
| 处理方法 | `convert()` | `convertNew()` |
