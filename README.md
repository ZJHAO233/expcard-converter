# ExpCard Converter

将试验卡 Excel 表格中的逻辑条件快速提取并转换为 Markdown 格式，支持导出为 Word 文档。

## 核心功能

**解决的问题：** 试验卡通常以 Excel 格式存储，包含复杂的逻辑条件（与/或/或取反），手动提取整理耗时且易出错。本工具自动识别并转换这些逻辑结构，生成结构化的 Markdown 文档。

### 主要特性

- **智能逻辑提取** - 自动识别"与"、"且"、"或"、"或取反"等逻辑运算符
- **多级结构支持** - 保持原文档的层级结构（二级标题、三级标题、列表项）
- **特殊分隔符处理** - 自动检测并转换"或延时"、"与延时"等特殊分隔符
- **实时预览** - 转换后即时预览 Markdown 效果
- **大纲导航** - 生成可折叠的大纲树，快速定位内容
- **批量处理** - 支持单个 Sheet 或全部 Sheet 批量转换
- **多格式导出** - 支持 Markdown 和 Word 文档导出
- **主题切换** - 支持浅色、深色、毛玻璃三种主题

## 快速开始

### 安装依赖

```bash
cd dcs-expcard-tool-nodejs
npm install
```

### 启动服务

```bash
npm start
```

服务器将在 `http://localhost:3210` 启动。

### 操作步骤

1. **上传文件** - 拖拽或点击上传区域，选择 `.xlsx` 格式的试验卡文件
2. **选择 Sheet** - 在左侧列表中选择要转换的工作表
3. **选择范围** - 选择转换"当前"Sheet 或"全部"Sheet
4. **执行转换** - 点击"转换"按钮
5. **预览结果** - 在右侧区域预览转换后的 Markdown 内容
6. **导出文件** - 点击"导出 Markdown"或"下载 Word"按钮

## 架构说明

### 1. 服务器层 (`server.js`)

基于 Express 框架，提供以下功能：

- **静态文件服务** - 提供前端页面访问
- **API 接口** - `/api/convert-to-word` 端点，接收 Markdown 内容并转换为 Word 文档
- **Pandoc 集成** - 调用 pandoc.exe 执行格式转换
- **临时文件管理** - 自动创建和清理临时文件

### 2. 转换引擎 (`convert.js`)

`ExpCardConverter` 类是核心转换逻辑，支持两种转换模式：

#### 数据处理流程

```
Excel 数据 → 行遍历 → 结构识别 → 逻辑提取 → Markdown 输出
```

#### 核心方法

| 方法 | 功能 |
|------|------|
| `convert(rows)` | 标准模式转换 |
| `convertNew(rows)` | 增强模式转换（支持更多层级） |
| `_processRow()` | 处理单行数据 |
| `_processLevel1()` | 处理一级条目（如 1. 2. 3.） |
| `_processLevel2()` | 处理二级条目（如 1.1 1.2） |
| `_processLevel3Group()` | 处理三级分组逻辑 |
| `_processLevel2Group()` | 处理二级分组逻辑 |

#### 识别的结构类型

| 结构 | 正则匹配 | 输出格式 |
|------|---------|---------|
| 中文数字标题 | `/^[一二三四五六七八九十]+、/` | `## 标题` |
| 子标题 | `/^\d+[\u4e00-\u9fa5]/` | `### 子标题` |
| 一级条目 | `/^\d+$/` | `1. 内容` |
| 二级条目 | `/^\d+\.\d+$/` | `   - 子内容` |

### 3. 配置系统 (`config.js`)

可自定义的配置项：

```javascript
const EXPCARD_CONFIG = {

  LOGIC_OPERATORS: {

};
```

### 4. 前端界面 (`index.html`)

单页应用，包含：

- **CSS 变量系统** - 支持三种主题（light/dark/glass）
- **响应式布局** - 适配桌面、平板、手机
- **文件处理** - 使用 SheetJS 读取 Excel
- **Markdown 渲染** - 自定义渲染器
- **大纲生成** - 从标题提取生成导航树

## 逻辑运算符处理

工具自动识别以下逻辑运算符并统一转换输出：

| Excel 中的值 | 输出格式 | 说明 |
|-------------|---------|------|
| 与 | 且 | 逻辑与 |
| 且 | 且 | 逻辑与（同义） |
| 或 | 或 | 逻辑或 |
| 或取反 | （或取反） | 逻辑或取反 |

## 特殊分隔符

以下特殊分隔符会被自动检测并转换：

| 特殊分隔符 | 转换为 |
|-----------|-------|
| 或延时 | 或 |
| 与延时 | 与 |

转换后，工具会在底部状态栏显示检测到的特殊分隔符位置，便于核对。

## 转换逻辑详解

### Excel 列定义

| 列 | 字段 | 用途 |
|---|------|------|
| A | 序号 | 编号，用于判断行的层级和类型 |
| B | 逻辑 | 逻辑运算符或第一层内容 |
| C | 内容 | 条件内容或逻辑分组标识 |
| D | 子组 | 子组逻辑运算符 |
| E | 详细 | 最终条件内容 |

---

### 行类型识别

根据 A 列的值判断行的类型：

| A 列内容 | 类型 | 处理方式 |
|---------|------|---------|
| `一、`、`二、`... | 中文数字标题 | 输出 `## 标题`，作为一级分隔 |
| `1.1 试验条件（与）` | 子标题 | 输出 `### 子标题`，提取括号内的逻辑运算符 |
| `1`、`2`、`3`... | 一级条目 | 输出 `1. 内容`，作为顶层条件 |
| `1.1`、`1.2`... | 二级子项 | 输出 `   - 子内容`，嵌套在一级条目下 |
| 空 | 继续行 | 延续上一行的层级关系 |
| `试验条件`、`结论`... | 段落标题 | 跳过，该部分内容不进行逻辑转换 |

---

### 两种转换模式

#### 旧模式 (`convert()`)

适用于 A 列使用 `x.y` 编号子项的试验卡。

**层级关系：**
- 中文数字标题（`一、`）→ `##` 二级标题
- 子标题（`1.1 试验条件（与）`）→ `###` 三级标题
- 一级条目（`1`、`2`）→ `1.` 有序列表
- 二级子项（`1.1`、`1.2`）→ `-` 无序列表

**逻辑运算符来源：**
- 子标题括号内的运算符（如 `试验条件（与）`）作为该标题下所有条目的默认运算符
- B 列的运算符用于一级条目之间的逻辑关系
- C 列触发分组开始，D 列触发子组开始

**注意事项：**
- 子标题必须包含括号内的逻辑运算符（如 `（与）`、`（或）`）
- 二级子项的编号必须严格遵循 `x.y` 格式
- 空行会被跳过，不会影响后续行的处理

---

#### 新模式 (`convertNew()`)

适用于 A 列使用纯数字编号、通过列嵌套表达层级的试验卡。

**层级关系（从外到内）：**
- B 列逻辑 → 最外层分组
- C 列逻辑 → 嵌套在 B 组内
- D 列逻辑 → 嵌套在 C 组内
- E 列 → 最终内容

**关键规则：**
- 每个 `###` 标题下编号从 1 重新开始
- B 列为空即视为该行是上一行的继续（无论 A 列值）
- A 列非空且 C 列有非逻辑内容时，视为独立新条目
- A 非空、B 空、C 空但 D 有内容时，视为 B 组的继续

**注意事项：**
- 新模式对列结构要求更严格，必须保证 B→C→D→E 的嵌套顺序
- 同一行内不能同时出现多个层级的逻辑运算符
- 编号重置只在遇到新的子标题时触发

---

### 逻辑运算符处理

**标准运算符映射：**

| Excel 值 | 输出值 | 说明 |
|---------|-------|------|
| 与 | 且 | 逻辑与 |
| 且 | 且 | 逻辑与（同义词） |
| 或 | 或 | 逻辑或 |
| 或取反 | （或取反） | 逻辑或取反，输出时加括号 |

**运算符优先级：** `（或取反）` > `且` > `或`

**处理规则：**
- 如果子标题已定义默认运算符，且条目运算符与之相同，则不重复显示
- 运算符缺失时，不添加逻辑标记
- 纯逻辑行（只有运算符没有内容）会被跳过

---

### 特殊分隔符处理

**特殊分隔符映射：**

| 特殊值 | 转换为 | 匹配方式 |
|-------|-------|---------|
| 或延时 | 或 | 前缀匹配 |
| 与延时 | 且 | 前缀匹配 |

**匹配规则：** 使用 `startsWith` 前缀匹配。例如 `或延时500ms` 会匹配 `或延时` 键，转换为 `或`，忽略后面的延时数值。

**状态显示：** 转换后，工具会在底部状态栏显示检测到的特殊分隔符位置（Sheet名、行号、列号），便于核对。

---

### 转换流程

1. 遍历每一行数据
2. 跳过空行和表头行
3. 根据 A 列判断行类型
4. 遇到段落标题时设置跳过标志
5. 遇到子标题时重置编号计数器
6. 根据行类型调用对应的处理方法
7. 处理逻辑运算符和特殊分隔符
8. 输出 Markdown 格式内容

---

### 注意事项

1. **空行处理：** 空行会被跳过，但可能影响行的继续判断
2. **纯逻辑行：** 只有运算符没有内容的行不会输出
3. **嵌套深度：** 支持最多 4 层嵌套，超过部分会合并
4. **特殊字符：** Excel 中的特殊字符会原样保留
5. **编号格式：** 一级条目必须是纯数字，二级子项必须是 `x.y` 格式
6. **运算符位置：** 运算符必须在 B 列或子标题括号内，其他位置无效
7. **段落标题：** 会跳过该标题下的所有内容，直到遇到下一个标题
8. **大小写：** 运算符区分中英文，只识别中文运算符

## 输出格式示例

```markdown
## 一、试验项目

### 1.1 试验条件（与）

1. 条件A（或）
   - 子条件1
   - 子条件2
2. 条件B（且）
   - 子条件3
```

## API 接口

### POST /api/convert-to-word

将 Markdown 内容转换为 Word 文档。

**请求体：**
```json
{
  "markdown": "# 标题\n\n内容...",
  "filename": "输出文件名"
}
```

**响应：**
- 成功：返回 `.docx` 文件二进制数据
- 失败：返回错误 JSON
