# ExpCard Converter (Node.js)

将试验卡 Excel 表格中的逻辑条件快速提取并转换为 Markdown 格式，支持导出为 Word 文档。

## 功能特性

- **智能逻辑提取** - 自动识别"与"、"且"、"或"、"或取反"等逻辑运算符
- **多级结构支持** - 保持原文档的层级结构
- **特殊分隔符处理** - 自动检测并转换"或延时"、"与延时"等特殊分隔符
- **实时预览** - 转换后即时预览 Markdown 效果
- **大纲导航** - 生成可折叠的大纲树，快速定位内容
- **批量处理** - 支持单个 Sheet 或全部 Sheet 批量转换
- **多格式导出** - 支持 Markdown 和 Word 文档导出
- **主题切换** - 支持浅色、深色、毛玻璃三种主题

## 快速开始

### 安装依赖

```bash
cd dcs-expcard-tool-nodejs
npm install
```

### 启动服务

```bash
npm start
```

服务器将在 `http://localhost:3210` 启动。

### 使用步骤

1. 上传 `.xlsx` 格式的试验卡文件
2. 选择要转换的工作表
3. 选择转换范围（当前 Sheet 或全部）
4. 点击"转换"按钮
5. 预览转换后的 Markdown 内容
6. 导出为 Markdown 或 Word 文档

## Excel 文件规范

### 文件格式要求

- 格式：`.xlsx`（不支持 `.xls`）
- 编码：UTF-8
- 首行：表头行（会被自动跳过）

---

### 两种处理模式

本工具支持两种 Excel 格式，根据 A 列的编号方式自动识别：

#### 模式一：子序号模式（x.y 格式）

A 列使用 `x.y` 格式的编号（如 `1.1`、`1.2`、`2.1`）。

**列结构：**

| 列 | 用途 | 示例 |
|---|------|------|
| A | 编号（必须是 x.y 格式） | `1.1`、`1.2` |
| B | 逻辑运算符 | `与`、`或` |
| C | 条件内容或分组标识 | `条件A`、`或` |
| D | 子组运算符 | `或`、`与` |
| E | 最终条件内容 | `子条件1` |

**Excel 示例：**

| A | B | C | D | E |
|---|---|---|---|---|
| 一、试验项目 | | | | |
| 1.1 试验条件（与） | | | | |
| 1 | 与 | 条件A | | |
| 1.1 | 或 | | 或 | 子条件1 |
| 1.2 | | | | 子条件2 |
| 2 | 或 | 条件B | | |

**转换结果：**

```markdown
## 一、试验项目

### 1.1 试验条件（与）

1. 条件A（与）
   - （或 子条件1 或 子条件2）
2. 条件B（或）
```

**特点：**
- A 列编号必须严格遵循 `x.y` 格式
- `x` 是一级条目编号，`y` 是二级子项编号
- 子标题格式：`1.1 试验条件（与）`（数字+中文+括号运算符）

---

#### 模式二：纯数字模式

A 列使用纯数字编号（如 `1`、`2`、`3`），通过列嵌套表达层级关系。

**列结构：**

| 列 | 用途 | 示例 |
|---|------|------|
| A | 编号（纯数字） | `1`、`2` |
| B | 第一层逻辑运算符 | `与`、`或` |
| C | 第二层逻辑运算符或内容 | `或`、`与` |
| D | 第三层逻辑运算符 | `或`、`与` |
| E | 最终条件内容 | `条件A` |

**层级关系（从外到内）：**
- B 列 → 最外层分组
- C 列 → 嵌套在 B 组内
- D 列 → 嵌套在 C 组内
- E 列 → 最终内容

**Excel 示例：**

| A | B | C | D | E |
|---|---|---|---|---|
| 一、试验项目 | | | | |
| 1.1 试验条件（与） | | | | |
| 1 | 与 | 或 | 或 | 条件A |
| | | | | 条件B |
| | | 与 | | 条件C |
| 2 | 或 | | 或 | 条件D |
| | | | | 条件E |

**转换结果：**

```markdown
## 一、试验项目

### 1.1 试验条件（与）

1. （或 条件A 或 条件B） 且 条件C（与）
2. （或 条件D 或 条件E）
```

**特点：**
- A 列编号是纯数字
- 层级通过 B→C→D→E 的列嵌套表达
- 空行表示延续上一行

---

### 两种模式对比

| 特征 | 子序号模式（x.y） | 纯数字模式 |
|------|------------------|-----------|
| A 列编号 | `1.1`、`1.2`、`2.1` | `1`、`2`、`3` |
| 层级表达 | 通过编号格式 | 通过列嵌套 |
| 子项关系 | 明确的父子编号 | 隐式的列层级 |
| 适用场景 | 结构清晰的试验卡 | 列式嵌套的试验卡 |

---

### 通用规范

#### 编号规则

**一级标题（A列）：**
- 格式：中文数字 + 顿号
- 示例：`一、`、`二、`、`三、`
- 作用：作为 `##` 二级标题输出

**子标题（A列）：**
- 格式：数字 + 中文 + 括号内运算符
- 示例：`1.1 试验条件（与）`、`2.1 试验内容（或）`
- 作用：作为 `###` 三级标题输出

#### 逻辑运算符规范

**位置：** 运算符只能出现在以下位置：
- 子标题的括号内（如 `试验条件（与）`）
- B 列

**支持的运算符：**

| 运算符 | 说明 | 输出 |
|-------|------|------|
| 与 | 逻辑与 | 且 |
| 且 | 逻辑与（同义） | 且 |
| 或 | 逻辑或 | 或 |
| 或取反 | 逻辑或取反 | （或取反） |

**注意：** 运算符必须是中文，英文运算符不识别。

#### 特殊分隔符规范

**支持的特殊分隔符：**

| 原始值 | 转换为 | 说明 |
|-------|-------|------|
| 或延时 | 或 | 忽略后面的延时数值 |
| 与延时 | 且 | 忽略后面的延时数值 |

**示例：** `或延时500ms` → `或`
