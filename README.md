# ExpCard Converter

试验卡数据提取工具，将 Excel 试验卡中的逻辑条件快速提取并转换为 Markdown 格式，支持导出为 Word 文档。

## 核心功能

- **智能逻辑提取** — 自动识别"与"、"且"、"或"、"或取反"等逻辑运算符
- **双模式转换** — 支持子序号模式（x.y 格式）和纯数字模式（列嵌套）
- **特殊分隔符处理** — 自动检测并转换"或延时"、"与延时"等特殊分隔符
- **Word 导出** — 通过 pandoc 将 Markdown 转换为 `.docx` 格式
- **多主题支持** — 浅色、深色、毛玻璃三种 UI 主题
- **大纲导航** — 可折叠的大纲树，快速定位内容

## 快速开始

### 方式一：使用启动脚本（推荐）

```
1. 右键 start.ps1 → 使用 PowerShell 运行
2. 选择 [1] 启动服务
3. 浏览器会自动打开
4. 拖入或点击上传 Excel 文件
5. 选择要转换的工作表
6. 点击「转换」按钮
7. 预览结果并导出
```

### 方式二：手动启动

```bash
cd dcs-expcard-tool-nodejs
npm install
npm start
```

服务启动后访问 `http://localhost:3210`

## 转换逻辑详解

### 整体流程

```
Excel 表格
    │
    ▼
┌─────────────────────────────────┐
│  1. 跳过表头行（序号、条件确认）   │
│  2. 识别中文数字标题（一、二、）   │
│  3. 识别段落标题（试验条件等）     │
│  4. 识别子标题（1.1 试验内容（与））│
│  5. 按模式处理数据行              │
└─────────────────────────────────┘
    │
    ▼
Markdown 输出
```

### 模式判断

| 条件 | 模式 | 说明 |
|------|------|------|
| A 列含 `x.y` 格式（如 `1.1`） | 子序号模式 | 调用 `convert()` |
| A 列为纯数字（如 `1`、`2`） | 纯数字模式 | 调用 `convertNew()` |

### 模式一：子序号模式（x.y 格式）

适用于 A 列使用 `1.1`、`1.2`、`2.1` 等编号的表格。

#### Excel 结构

| A (编号) | B (逻辑) | C (内容/逻辑) | D (子逻辑) | E (内容) |
|----------|----------|---------------|------------|----------|
| 试验内容 | | | | |
| 1.1 A磨润滑油条件满足（与） | | | | |
| | 与 | 条件A | | |
| | | 或 | | 子条件1 |
| | | | 或 | 子条件2 |
| | 或 | 条件B | | |

#### 处理流程

```
1.2 子标题行 "1.1 A磨润滑油条件满足（与）"
    │  提取括号内逻辑（与/或）→ 存为 currentSubTitleLogic
    │
    ▼
Level 1: 主条件行（1.2.1 的父级）
    │  输出: "1.2.1 A磨润滑油条件满足（且）"
    │  括号内的逻辑 = 下一行 B 列的逻辑
    │
    ▼
Level 2: 子条件行（1.2.1 的子级）
    │  检查 C 列是否为逻辑运算符
    │  ├─ 是 → 进入 Level 3 分组
    │  └─ 否 → 直接输出: "   - 条件A"
    │
    ▼
Level 3: 嵌套分组（C 列 + D 列都是逻辑运算符）
    │  按 D 列逻辑分组，E 列内容合并
    │  同组用逻辑连接，多组用"或"连接
    │  输出: "   - （子条件1 或 子条件2）"
```

#### 输出示例

```markdown
## 一、试验内容

### 1.1 A磨润滑油条件满足（且）

1.2.1 A磨润滑油条件满足（且）
   - 条件A 或 子条件1 或 子条件2
   - 条件B
```

### 模式二：纯数字模式

适用于 A 列使用纯数字 `1`、`2`、`3`，通过列嵌套表达层级关系。

#### Excel 结构

| A (编号) | B (逻辑) | C (逻辑/内容) | D (逻辑/内容) | E (内容) |
|----------|----------|---------------|---------------|----------|
| 试验内容 | | | | |
| 1 | 与 | 条件A | | |
| | | 或 | | 子条件1 |
| | | | 或 | 子条件2 |
| | 或 | 条件B | | |

#### 处理流程

```
纯数字行 "1"（A 列）
    │
    ├─ 下一行是 x.y 格式？→ 按子序号模式处理
    │
    └─ B 列是逻辑运算符？
        ├─ 否 → 输出: "1. 条件A"
        └─ 是 → 进入 Level 2
            │
            ▼
Level 2: B 列逻辑 + C 列判断
    │  C 列是逻辑运算符？
    │  ├─ 是 → 进入 Level 3
    │  └─ 否 → 收集后续行的 C 列内容，用 B 列逻辑连接
    │      输出: "1. 条件A 或 子条件1 或 子条件2"
    │
    ▼
Level 3: C 列逻辑 + D 列判断
    │  D 列是逻辑运算符？
    │  ├─ 是 → 进入 Level 4
    │  └─ 否 → 收集后续行的 D 列内容，用 C 列逻辑连接
    │      多项用括号包裹: "（子条件1 或 子条件2）"
    │
    ▼
Level 4: D 列逻辑 + E 列内容
    收集后续行的 E 列内容，用 D 列逻辑连接
    多项用括号包裹: "（子条件1 或 子条件2）"
```

#### 输出示例

```markdown
## 一、试验内容

1. 条件A 或 （子条件1 或 子条件2）
2. 条件B
```

### 逻辑运算符映射

| Excel 值 | 输出值 | 说明 |
|---------|-------|------|
| 与 | 且 | 逻辑与 |
| 且 | 且 | 逻辑与（同义词） |
| 或 | 或 | 逻辑或 |
| 或取反 | （或取反） | 逻辑或取反，输出时加括号 |

### 特殊分隔符

采用**前缀匹配**，用于处理"或延时xxS"等无法正常拼接的逻辑。

| 特殊值 | 转换为 | 示例 |
|-------|-------|------|
| 或延时 | 或 | `或延时720s` → `或` |
| 与延时 | 且 | `与延时300s` → `且` |

### 跳过规则

以下行会被跳过，不参与转换：

- 表头行：`序号`、`条件确认`
- 段落标题：`试验条件`、`试验恢复`、`结论`、`存在问题`
- `试验条件` 之后、`试验内容` 之前的所有行

### 嵌套括号规则

| 层级 | 条件 | 输出格式 |
|------|------|---------|
| Level 2 单项 | 无分组 | `- 内容` |
| Level 2 分组 | C 列是逻辑运算符 | `- （内容1 或 内容2）` |
| Level 3 | D 列是逻辑运算符 | `- （（内容1 或 内容2） 或 内容3）` |
| Level 4 | E 列多行 | `- （（（内容1 或 内容2） 或 内容3） 或 内容4）` |

## 配置文件

配置文件 `config.js` 位于程序目录，可在启动前修改。

```javascript
const EXPCARD_CONFIG = {
  // 逻辑运算符映射
  LOGIC_OPERATORS: {
    "与": "且",
    "且": "且",
    "或": "或",
    "或取反": "或取反",
  },

  // 特殊分隔符映射（前缀匹配）
  SPECIAL_SEPARATORS: {
    "或延时": "或",
    "与延时": "且",
  },

  // 跳过的表头
  SKIP_HEADERS: ["序号", "条件确认"],

  // 段落标题（不进行逻辑转换）
  SECTION_HEADERS: ["试验条件", "试验恢复", "结论", "存在问题"],
};
```

保存后需重启服务生效。

## Excel 文件规范

### 格式要求

- 格式：`.xlsx`（不支持 `.xls`）
- 编码：UTF-8
- 首行：表头行（自动跳过）

### 两种模式对比

| 特性 | 子序号模式 | 纯数字模式 |
|------|-----------|-----------|
| A 列编号 | `x.y` 格式（如 `1.1`） | 纯数字（如 `1`） |
| 层级表达 | 编号本身表达层级 | 列嵌套表达层级 |
| 适用场景 | 层级清晰的表格 | 宽表格式 |

## 技术栈

- **前端**：纯 HTML/CSS/JS，xlsx.js（Excel 解析）、marked.js（Markdown 渲染）
- **后端**：Express.js（Node.js）
- **转换**：Pandoc（Markdown → Word）
- **打包**：SEA（Single Executable Application）
- **启动**：PowerShell 脚本
