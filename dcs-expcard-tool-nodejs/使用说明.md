# ExpCard Converter 使用说明

## 快速启动

### 方式一：双击启动

双击 `start.bat` 文件，等待命令行窗口显示"服务器已启动"后，在浏览器中访问：

```
http://localhost:3210
```

### 方式二：命令行启动

```bash
node server.js
```

---

## 基本使用步骤

1. 打开浏览器访问 `http://localhost:3210`
2. 拖拽或点击上传 `.xlsx` 格式的试验卡文件
3. 在左侧选择要转换的工作表（Sheet）
4. 选择转换范围：「当前」或「全部」
5. 点击「转换」按钮
6. 在右侧预览转换结果
7. 点击「导出 Markdown」或「下载 Word」保存文件

---

## 配置文件说明

配置文件 `config.js` 位于程序目录，可在启动前修改。

### 修改逻辑运算符

打开 `config.js`，找到 `LOGIC_OPERATORS` 部分：

```javascript
LOGIC_OPERATORS: {
  "与": "且",      // Excel中的"与" → 输出"且"
  "且": "且",      // Excel中的"且" → 输出"且"
  "或": "或",      // Excel中的"或" → 输出"或"
  "或取反": "或取反",  // Excel中的"或取反" → 输出"或取反"
  // 在此添加新的映射
}
```

### 修改特殊分隔符

找到 `SPECIAL_SEPARATORS` 部分：

```javascript
SPECIAL_SEPARATORS: {
  "或延时": "或",  // Excel中的"或延时" → 输出"或"
  "与延时": "且",  // Excel中的"与延时" → 输出"且"
  // 在此添加新的映射
}
```

### 修改后生效

保存 `config.js` 后，需要重启服务才能生效。

## Excel 文件规范

### 文件格式要求

- 格式：`.xlsx`（不支持 `.xls`）
- 编码：UTF-8
- 首行：表头行（会被自动跳过）

---

### 列结构

每列的固定用途如下：

| 列 | 用途 | 示例 |
|---|------|------|
| **A** | 编号 | `一、`、`1`、`1.1` |
| **B** | 逻辑运算符 | `与`、`或`、`或取反` |
| **C** | 条件内容或分组 | `条件A`、`或`、`与` |
| **D** | 子组运算符 | `或`、`与` |
| **E** | 最终条件内容 | `子条件1`、`子条件2` |

---

### 编号规则

**一级标题（A列）：**
- 格式：中文数字 + 顿号
- 示例：`一、`、`二、`、`三、`
- 作用：作为 `##` 二级标题输出

**子标题（A列）：**
- 格式：数字 + 中文 + 括号内运算符
- 示例：`1.1 试验条件（与）`、`2.1 试验内容（或）`
- 作用：作为 `###` 三级标题输出

**一级条目（A列）：**
- 格式：纯数字
- 示例：`1`、`2`、`3`
- 作用：作为 `1.` 有序列表项

**二级子项（A列）：**
- 格式：数字.数字
- 示例：`1.1`、`1.2`、`2.1`
- 作用：作为 `   -` 无序列表项

---

### 逻辑运算符规范

**位置：** 运算符只能出现在以下位置：
- 子标题的括号内（如 `试验条件（与）`）
- B 列

**支持的运算符：**

| 运算符 | 说明 | 输出 |
|-------|------|------|
| 与 | 逻辑与 | 且 |
| 且 | 逻辑与（同义） | 且 |
| 或 | 逻辑或 | 或 |
| 或取反 | 逻辑或取反 | （或取反） |

**注意：** 运算符必须是中文，英文运算符不识别。

---

### 特殊分隔符规范

**支持的特殊分隔符：**

| 原始值 | 转换为 | 说明 |
|-------|-------|------|
| 或延时 | 或 | 忽略后面的延时数值 |
| 与延时 | 且 | 忽略后面的延时数值 |

**示例：** `或延时500ms` → `或`

---

### 标准 Excel 模板

```
┌─────────┬─────────┬─────────┬─────────┬─────────┐
│    A    │    B    │    C    │    D    │    E    │
├─────────┼─────────┼─────────┼─────────┼─────────┤
│ 一、试验项目 │         │         │         │         │
├─────────┼─────────┼─────────┼─────────┼─────────┤
│ 1.1 试验条件（与）│         │         │         │         │
├─────────┼─────────┼─────────┼─────────┼─────────┤
│    1    │    与    │  条件A   │         │         │
├─────────┼─────────┼─────────┼─────────┼─────────┤
│   1.1   │    或    │         │    或    │ 子条件1   │
├─────────┼─────────┼─────────┼─────────┼─────────┤
│   1.2   │         │         │         │ 子条件2   │
├─────────┼─────────┼─────────┼─────────┼─────────┤
│    2    │    或    │  条件B   │         │         │
└─────────┴─────────┴─────────┴─────────┴─────────┘
```

**转换结果：**

```markdown
## 一、试验项目

### 1.1 试验条件（与）

1. 条件A（与）
   - （或 子条件1 或 子条件2）
2. 条件B（或）
```

## 常见问题

### Q: 上传文件后没有显示 Sheet 列表？

**A:** 检查文件是否为 `.xlsx` 格式，`.xls` 格式不支持。

### Q: 转换结果为空？

**A:** 检查 Excel 文件是否包含以下内容：
- 中文数字标题（如 `一、`）
- 或子标题（如 `1.1 试验条件（与）`）

### Q: 逻辑运算符没有被识别？

**A:** 确认运算符是否为中文，且位于正确的列：
- 子标题括号内的运算符
- B 列的运算符

### Q: 导出 Word 失败？

**A:** 检查程序目录下是否存在 `pandoc.exe` 文件。

### Q: 如何添加新的逻辑运算符？

**A:** 编辑 `config.js` 文件，在 `LOGIC_OPERATORS` 中添加映射，保存后重启服务。

### Q: 如何修改特殊分隔符？

**A:** 编辑 `config.js` 文件，在 `SPECIAL_SEPARATORS` 中添加映射，保存后重启服务。

## 注意事项

1. **文件格式**：必须使用 `.xlsx` 格式，不支持 `.xls`
2. **运算符识别**：只识别中文运算符，英文运算符无效
3. **编号规范**：一级条目必须是纯数字，二级子项必须是 `x.y` 格式
4. **配置修改**：修改 `config.js` 后必须重启服务
5. **Word 导出**：需要 `pandoc.exe` 文件在程序目录下
6. **浏览器兼容**：建议使用 Chrome、Firefox、Edge 等现代浏览器
7. **端口占用**：默认端口 3210，如被占用可修改 `server.js` 中的 `PORT` 变量

## 端口修改

如需修改端口，编辑 `server.js`，找到以下行：

```javascript
const PORT = process.env.PORT || 3210;
```

将 `3210` 改为所需端口号，保存后重启服务。
