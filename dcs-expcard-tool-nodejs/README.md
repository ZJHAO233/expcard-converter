# ExpCard Converter (Node.js 版)

将试验卡 Excel 表格中的逻辑条件快速提取并转换为 Markdown 格式，并支持导出为 Word 文档。

## 核心功能

**解决的问题：** 试验卡通常以 Excel 格式存储，包含复杂的逻辑条件（与/或/或取反），手动提取整理耗时且易出错。本工具自动识别并转换这些逻辑结构，生成结构化的 Markdown 文档，并可通过 pandoc 导出为 Word。

### 主要特性

- **智能逻辑提取** — 自动识别"与"、"且"、"或"、"或取反"等逻辑运算符
- **双模式转换** — 支持"旧模式"（编号子项 `x.y`）和"新模式"（列嵌套 B→C→D→E）
- **特殊分隔符处理** — 自动检测并转换"或延时"、"与延时"等特殊分隔符
- **Word 导出** — 通过 pandoc 将 Markdown 转换为 `.docx` 格式
- **多主题支持** — 浅色、深色、毛玻璃三种 UI 主题
- **大纲导航** — 可折叠的大纲树，快速定位内容

## 快速开始

### 安装依赖

```bash
cd dcs-expcard-tool-nodejs
npm install
```

### 启动服务

```bash
npm start
```

### 使用启动脚本（推荐）

```bash
# Windows
start.bat

# 或者手动启动
node server.js
```

服务启动后访问 `http://localhost:3210`

### 操作步骤

1. **上传文件** — 拖拽或点击上传区域，选择 `.xlsx` 格式的试验卡文件
2. **选择 Sheet** — 在左侧列表中选择要转换的工作表
3. **选择范围** — 选择转换"当前"Sheet 或"全部"Sheet
4. **选择模式** — 选择"旧模式"（`x.y` 编号子项）或"新模式"（列嵌套）
5. **执行转换** — 点击"转换"按钮
6. **预览结果** — 在右侧区域预览转换后的 Markdown 内容
7. **导出文件** — 点击"导出 Markdown"下载 `.md` 文件，或点击"导出 Word"下载 `.docx` 文件

## 配置文件说明

项目支持外部配置文件 `config.js`，可在启动前修改。

### 生成默认配置

```bash
npm run init-config
```

### 配置文件结构

```javascript
const EXPCARD_CONFIG = {
  // 逻辑运算符映射
  LOGIC_OPERATORS: {
    "与": "且",    // Excel中的"与" → 输出"且"
    "且": "且",    // Excel中的"且" → 输出"且"
    "或": "或",    // Excel中的"或" → 输出"或"
    "或取反": "或取反",  // Excel中的"或取反" → 输出"或取反"
  },

  // 特殊分隔符映射
  SPECIAL_SEPARATORS: {
    "或延时": "或",  // Excel中的"或延时" → 输出"或"
    "与延时": "且",  // Excel中的"与延时" → 输出"且"
  },

  // 跳过的表头（不处理的行）
  SKIP_HEADERS: ["序号", "条件确认"],

  // 段落标题（这些标题下的内容不进行逻辑转换）
  SECTION_HEADERS: ["试验条件", "试验恢复", "结论", "存在问题"],
};
```

### 添加新的逻辑运算符


### 添加新的特殊分隔符

在 `config.js` 的 `SPECIAL_SEPARATORS` 对象中添加映射：

## 转换逻辑详解

### Excel 列定义

| 列 | 字段 | 用途 |
|---|------|------|
| A | 序号 | 编号，用于判断行的层级和类型 |
| B | 逻辑 | 逻辑运算符或第一层内容 |
| C | 内容 | 条件内容或逻辑分组标识 |
| D | 子组 | 子组逻辑运算符 |
| E | 详细 | 最终条件内容 |

---

### 行类型识别

根据 A 列的值判断行的类型：

| A 列内容 | 类型 | 处理方式 |
|---------|------|---------|
| `一、`、`二、`... | 中文数字标题 | 输出 `## 标题`，作为一级分隔 |
| `1.1 试验条件（与）` | 子标题 | 输出 `### 子标题`，提取括号内的逻辑运算符 |
| `1`、`2`、`3`... | 一级条目 | 输出 `1. 内容`，作为顶层条件 |
| `1.1`、`1.2`... | 二级子项 | 输出 `   - 子内容`，嵌套在一级条目下 |
| 空 | 继续行 | 延续上一行的层级关系 |
| `试验条件`、`结论`... | 段落标题 | 跳过，该部分内容不进行逻辑转换 |

---

### 两种转换模式

#### 旧模式 (`convert()`)

适用于 A 列使用 `x.y` 编号子项的试验卡。

**层级关系：**
- 中文数字标题（`一、`）→ `##` 二级标题
- 子标题（`1.1 试验条件（与）`）→ `###` 三级标题
- 一级条目（`1`、`2`）→ `1.` 有序列表
- 二级子项（`1.1`、`1.2`）→ `-` 无序列表

**逻辑运算符来源：**
- 子标题括号内的运算符（如 `试验条件（与）`）作为该标题下所有条目的默认运算符
- B 列的运算符用于一级条目之间的逻辑关系
- C 列触发分组开始，D 列触发子组开始

**注意事项：**
- 子标题必须包含括号内的逻辑运算符（如 `（与）`、`（或）`）
- 二级子项的编号必须严格遵循 `x.y` 格式
- 空行会被跳过，不会影响后续行的处理

---

#### 新模式 (`convertNew()`)

适用于 A 列使用纯数字编号、通过列嵌套表达层级的试验卡。

**层级关系（从外到内）：**
- B 列逻辑 → 最外层分组
- C 列逻辑 → 嵌套在 B 组内
- D 列逻辑 → 嵌套在 C 组内
- E 列 → 最终内容

**关键规则：**
- 每个 `###` 标题下编号从 1 重新开始
- B 列为空即视为该行是上一行的继续（无论 A 列值）
- A 列非空且 C 列有非逻辑内容时，视为独立新条目
- A 非空、B 空、C 空但 D 有内容时，视为 B 组的继续

**注意事项：**
- 新模式对列结构要求更严格，必须保证 B→C→D→E 的嵌套顺序
- 同一行内不能同时出现多个层级的逻辑运算符
- 编号重置只在遇到新的子标题时触发

---

### 逻辑运算符处理

**标准运算符映射：**

| Excel 值 | 输出值 | 说明 |
|---------|-------|------|
| 与 | 且 | 逻辑与 |
| 且 | 且 | 逻辑与（同义词） |
| 或 | 或 | 逻辑或 |
| 或取反 | （或取反） | 逻辑或取反，输出时加括号 |

**运算符优先级：** `（或取反）` > `且` > `或`

**处理规则：**
- 如果子标题已定义默认运算符，且条目运算符与之相同，则不重复显示
- 运算符缺失时，不添加逻辑标记
- 纯逻辑行（只有运算符没有内容）会被跳过

---

### 特殊分隔符处理

**特殊分隔符映射：**

| 特殊值 | 转换为 | 匹配方式 |
|-------|-------|---------|
| 或延时 | 或 | 前缀匹配 |
| 与延时 | 且 | 前缀匹配 |

**匹配规则：** 使用 `startsWith` 前缀匹配。例如 `或延时500ms` 会匹配 `或延时` 键，转换为 `或`，忽略后面的延时数值。

**状态显示：** 转换后，工具会在底部状态栏显示检测到的特殊分隔符位置（Sheet名、行号、列号），便于核对。

---

### 转换流程

1. 遍历每一行数据
2. 跳过空行和表头行
3. 根据 A 列判断行类型
4. 遇到段落标题时设置跳过标志
5. 遇到子标题时重置编号计数器
6. 根据行类型调用对应的处理方法
7. 处理逻辑运算符和特殊分隔符
8. 输出 Markdown 格式内容

---

### 注意事项

1. **空行处理：** 空行会被跳过，但可能影响行的继续判断
2. **纯逻辑行：** 只有运算符没有内容的行不会输出
3. **嵌套深度：** 支持最多 4 层嵌套，超过部分会合并
4. **特殊字符：** Excel 中的特殊字符会原样保留
5. **编号格式：** 一级条目必须是纯数字，二级子项必须是 `x.y` 格式
6. **运算符位置：** 运算符必须在 B 列或子标题括号内，其他位置无效
7. **段落标题：** 会跳过该标题下的所有内容，直到遇到下一个标题
8. **大小写：** 运算符区分中英文，只识别中文运算符

## 配置文件

`config.js` 包含所有可配置的字典，用户可自由修改：

```javascript
const EXPCARD_CONFIG = {
  // 逻辑运算符字典
  LOGIC_OPERATORS: {
    与: "且",
    且: "且",
    或: "或",
    或取反: "或取反",
  },

  // 特殊分隔符字典
  SPECIAL_SEPARATORS: {
    或延时: "或",
    与延时: "且",
  },

  // 跳过的表头
  SKIP_HEADERS: ["序号", "条件确认"],

  // 段落标题
  SECTION_HEADERS: ["试验条件", "试验恢复", "结论", "存在问题"],
};
```

- `LOGIC_OPERATORS`：逻辑运算符映射（键为 Excel 值，值为输出格式）
- `SPECIAL_SEPARATORS`：特殊分隔符映射（前缀匹配）
- `SKIP_HEADERS`：需要跳过的表头行
- `SECTION_HEADERS`：段落标题（不参与转换的固定标题）

## API 接口

### POST /api/convert-to-word

将 Markdown 内容转换为 Word 文档。

**请求体：**
```json
{
  "markdown": "## 一、试验项目\n...",
  "filename": "试验卡"
}
```

**响应：** `.docx` 文件流（Content-Type: `application/vnd.openxmlformats-officedocument.wordprocessingml.document`）

**错误处理：**
- 400：Markdown 内容为空
- 500：Pandoc 转换失败

## 技术栈

- **前端**：纯 HTML/CSS/JS，xlsx.js（Excel 解析）、marked.js（Markdown 渲染）、JSZip
- **后端**：Express.js（Node.js）
- **转换**：Pandoc（Markdown → Word）
- **打包**：支持 pkg 编译为独立 exe
